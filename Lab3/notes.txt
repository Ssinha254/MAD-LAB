ViewPager is an Android UI component that lets users swipe left and right between screens (pages).

Think of it like:

Instagram stories

Tabs you can swipe between

Pages in a book 

Each swipe shows a new Fragment.
Example use cases:

Tabs (Home / Profile / Settings)

Onboarding screens

Image sliders

An Adapter is the middleman.

 It tells the ViewPager:

How many pages there are

What Fragment to show for each page

Without an adapter, ViewPager has no idea what to display.

Think of it like:

ViewPager = bookshelf
Adapter = librarian who brings the right book for each shelf
What is FragmentPagerAdapter?

FragmentPagerAdapter is a special adapter made for ViewPager that works with Fragments.

It:

Creates fragments

Keeps them in memory (good for small number of pages)

 Note: In newer Android versions, this is replaced by
ViewPager2 + FragmentStateAdapter
What is TabFragment?

That’s your custom Fragment.

Your MainActivity is declared as:

So at runtime Android goes:

“Looking for com.maverick.myapplication.MainActivity…
Found nothing. Crash.”

✅ Two correct ways to fix it (pick ONE)
🟢 Option 1 — Change the package of MainActivity (most common)

Update MainActivity.kt to:

package com.maverick.myapplication


✔ Easiest
✔ Clean
✔ Best if this is a small project

Then:

Build → Clean Project
Build → Rebuild Project

2️⃣ Difference between onCreate() and onCreateView()
Method	Class	Purpose	Key Points
onCreate(Bundle savedInstanceState)	Activity	Activity initialization	Set content view, find views, set adapters, attach listeners
onCreateView(LayoutInflater, ViewGroup, Bundle)	Fragment	Inflate Fragment’s UI	Return the fragment’s view hierarchy (XML)
Key points:You cannot inflate fragment UI in onCreate() → use onCreateView()

You cannot call findViewById() for fragment layout in Activity.onCreate() → must do it in fragment view

What does inflate mean?
return inflater.inflate(R.layout.fragment_top_stories, container, false);


inflater → A tool that turns XML layout into actual View objects in memory.

R.layout.fragment_top_stories → The XML layout file for this fragment.

container → The parent ViewGroup in which this fragment will live (usually ViewPager2 or Activity layout).

false → Don’t attach to parent yet; system will do it.

✅ In short:

“inflate” = load XML and turn it into a screen you can see and interact with

3️⃣ What is position in createFragment(int position)?
switch (position) {
    case 0:
        return new TopStoriesFragment();
    case 1:
        return new SportsFragment();
    case 2:
        return new EntertainmentFragment();
}


position = index of the page the ViewPager2 is currently asking for.

0 → first tab

1 → second tab

2 → third tab

ViewPager2 calls createFragment(position) every time it needs a fragment for a particular page.

4️⃣ What is new TabLayoutMediator(...) doing?
new TabLayoutMediator(tabLayout, viewPager,
    (tab, position) -> {
        switch (position) {
            case 0:
                tab.setText("Top Stories");
                break;
            case 1:
                tab.setText("Sports");
                break;
            case 2:
                tab.setText("Entertainment");
                break;
        }
    }).attach();


TabLayoutMediator = Connects tabs with ViewPager2.

tab → Each tab object

position → Index of tab (0,1,2)

tab.setText() → Sets the text shown on the tab

.attach() → Actually links tabs with pages

✅ In short:

It tells the system: “Tab 0 = Top Stories, Tab 1 = Sports… and make tabs swipeable”

5️⃣ Will it return a new fragment each time?

Yes, createFragment() creates a new fragment object.

But ViewPager2 keeps fragments in memory (default offscreen page limit = 1)

So if you swipe back to an old fragment, the same fragment instance is usually reused (ViewPager2 manages this automatically)

If you scroll too far, old fragments may be destroyed and recreated to save memory.

6️⃣ Why do we write @Override?
@Override
public Fragment createFragment(int position)


@Override tells the compiler:

“I am overriding a method from the parent class”

Compiler checks if method exists in parent; if not, gives error → avoids mistakes

Makes code cleaner and safer

✅ In short:

It’s a safety tag, not mandatory but highly recommended
